Case Study 01

Bolt Builders Initiative

Launched community program during Bolt.new's hypergrowth phase. Rapid iteration, ruthless prioritization, learning velocity over perfection.

The Challenge

Bolt.new was scaling explosively but lacked structured engagement with power users. Needed a way to gather product insights, validate new capabilities, and build community at speed.

The Prototype

Pitched and launched Bolt Builders - a simple Slack community with weekly showcases. No complex tech. Just fast execution focused on learning.

  • Slack community infrastructure set up in days
  • Weekly showcase format for rapid feedback loops
  • Direct connection between builders and product team
  • Led to world's largest AI hackathon (thousands of participants)

What I Learned & Discarded

Learned: Builders didn't need more features - they needed better examples and peer learning. Community-driven validation beats traditional research. Speed of feedback loop matters more than sophistication of tooling.

Discarded: Complex onboarding flows, elaborate documentation systems, formal program structure. Kept it simple, focused on high-signal interactions.

Impact

  • Informed product decisions worth millions
  • Community-driven growth became primary acquisition channel
  • Initiative became official company program
  • Direct insights from frontier usage patterns

Case Study 02

YachtOS: Voice-First AI Captain's Assistant

Multi-modal AI agent built with Claude Sonnet 4.5, Whisper, ElevenLabs, and vision AI. Rapid prototype demonstrating emerging capabilities at the frontier.

The Prototype

Voice-first application for superyacht operations. Natural language commands processed through AI agent with 7 specialized tools. Vision AI for maintenance photo analysis. Built to validate multi-modal frontier capabilities.

  • Next.js 14 + Vercel AI SDK for agent orchestration
  • Claude Sonnet 4.5 for reasoning and decision-making
  • Whisper (voice-to-text) + ElevenLabs (text-to-voice)
  • Vision AI analyzing maintenance photos, identifying parts and issues
  • 7 AI tools: weather, vessel tracking, work orders, supplier search, scheduling
  • Full-stack application shipped in under a week

Frontier Capabilities Validated

This prototype demonstrated emerging AI capabilities in real-world context: multi-modal inputs (voice + vision), agentic workflows with tool use, streaming responses, and context-aware reasoning. Built to learn what works at the frontier, not to achieve perfection.

What I Learned

  • Multi-modal AI needs careful UX design - voice + vision require different interaction patterns
  • Agent tool reliability matters more than sophistication - 7 reliable tools beat 20 flaky ones
  • Streaming is critical for user experience in agentic workflows
  • Real-world domain knowledge (yacht operations) shapes how you architect AI capabilities

Case Study 03

Chilled Sites: AI Development Platform

Built complete AI coding platform as solo founder. Competes with Bolt, Lovable, and Replit. Multi-model support, prompt-to-code generation, MCP servers in production.

The Prototype (That Became a Product)

Started as rapid prototype, evolved into full platform. React app generation, website building, MCP server integration, asset generation - all orchestrated through AI. Solo-built the entire architecture.

  • Multi-model: Claude (3.7, 4, 4.5), GPT-4o, o1, Gemini, Grok
  • Prompt-to-code: users describe vision, AI builds complete applications
  • Real-time AI editing: inline code modifications using LLMs
  • MCP servers: database queries, asset generation, analytics during generation
  • V2 prompt system with agent specialization
  • 30+ SEO-optimized pages, PWA generation, business analytics dashboard

Iteration & Learning

Countless prototypes discarded. Tested different model orchestration approaches, prompt architectures, code generation pipelines. Current V2 system emerged from ruthless iteration - threw out entire V1 architecture when better approach became clear.

Frontier Insights

  • Model selection matters by task - Claude excels at reasoning, GPT-4o at structured output
  • Streaming UX is critical - users need visibility into AI's work
  • MCP architecture enables capabilities impossible with simple prompting
  • Prompt engineering at scale requires systematic architecture, not one-off tricks
